# QA Notes

Testing notes for experienced QA. No theory, only practice.

## Structure

Each note should include:
- **Edge cases**: Non-obvious scenarios
- **Gotchas**: Easy mistakes
- **Constraints**: Important limits
- **Dependencies**: What must work first

## Naming

`{feature-or-task}.md` — e.g., `learning-loop.md`, `hypothesis-creation.md`
