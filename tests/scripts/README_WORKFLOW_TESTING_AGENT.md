# Workflow Testing Agent — Error-Fix Loop

**Версия:** v1.0  
**Назначение:** Агент автоматически тестирует n8n workflows с автоматическим исправлением ошибок в цикле до успеха

## 🎯 Что это такое?

**Error-Fix Loop** — это автоматический процесс тестирования и исправления workflows:

```
Тестирование → Ошибка? → Исправление → Тестирование → ... → Успех!
```

Агент автоматически:
1. ✅ Тестирует workflow
2. ✅ Анализирует ошибки
3. ✅ Исправляет ошибки автоматически
4. ✅ Тестирует заново
5. ✅ Повторяет до успеха (максимум 10 итераций)

## 🚀 Быстрый старт

### В Cursor (автоматически):

**Просто скажи агенту:**

```
"Протестируй workflow из Issue #22"
"Исправь и протестируй workflow"
"Запусти error-fix loop для Epic #23"
```

**Агент автоматически:**
1. Прочитает Issue/Epic через GitHub API
2. Извлечёт Workflow ID(ы) из Issue body
3. Запустит error-fix loop
4. Покажет результаты в Issue

### Ручной запуск:

```bash
# Тестирование одного workflow
node tests/scripts/fix_and_test_workflow.js 22

# Тестирование с явным Workflow ID
node tests/scripts/fix_and_test_workflow.js 22 cGSyJPROrkqLVHZP

# Тестирование всех workflows из Epic
node tests/scripts/execute_task_block.js 23

# Тестирование конкретных Issues
node tests/scripts/execute_task_block.js 24 25 26 27
```

## 🛠️ Типы исправляемых ошибок

### ✅ Автоматически исправляются:

1. **Schema Validation**
   - Добавление `useCustomSchema: true`
   - Добавление `schema: "genomai"` для Supabase nodes

2. **Missing Parameters**
   - Автоматическое определение `tableId` из имени node
   - Добавление обязательных параметров

3. **Database Errors**
   - Исправление проблем с schema
   - Исправление `tableId` (объект → выражение)

### ⚠️ Требуют ручного исправления:

1. **Connection Errors**
   - Проблемы с credentials
   - Проблемы с подключением к базе данных

2. **Expression Errors**
   - Синтаксические ошибки в выражениях
   - Неправильные ссылки на переменные

3. **Unknown Errors**
   - Неизвестные типы ошибок
   - Сложные логические ошибки

## 📋 Требования

### Переменные окружения:

```bash
# Обязательные
export GITHUB_TOKEN="your-github-token"
export N8N_API_KEY="your-n8n-api-key"

# Опциональные
export MAX_ITERATIONS=10        # Максимум итераций (по умолчанию: 10)
export AUTO_FIX=false            # Отключить автоматическое исправление
export N8N_API_URL="https://..." # URL n8n API (по умолчанию: из env)
```

### Формат Issue:

Issue должен содержать Workflow ID в формате:

```markdown
**Workflow ID:** `workflow-id`
```

или

```markdown
Workflow ID: `workflow-id`
```

## 🎯 Использование в Cursor

### Автоматическая активация:

Агент автоматически активируется когда:

1. **Issue содержит "Testing & Validation"** в названии или body
2. **Issue содержит Workflow ID** в правильном формате
3. **Epic содержит Issues с тестированием** — автоматически тестирует все workflows
4. **Пользователь явно просит:** "Протестируй workflow", "Исправь и протестируй"

### Примеры команд:

```
"Протестируй workflow из Issue #22"
"Исправь и протестируй workflow"
"Запусти error-fix loop для Epic #23"
"Тестируй все workflows из Epic #23"
```

## 📊 Результаты

После завершения цикла агент:

1. **Добавляет комментарий в Issue** с результатами:
   - Количество итераций
   - Статус (успешно/требует ручного исправления)
   - Список исправленных ошибок
   - Ссылка на workflow

2. **Обновляет Issue** (если успешно):
   - Отмечает чеклист как выполненный
   - Обновляет статус

3. **Логирует результаты** в консоль:
   - Детальный отчёт по каждой итерации
   - Список исправленных ошибок
   - Финальный статус

## 🔍 Troubleshooting

### Ошибка: "Не удалось автоматически исправить"

**Причины:**
- Ошибка требует ручного исправления (connection, expression, unknown)
- Достигнут максимум итераций (10)

**Решение:**
- Проверьте ошибки в последней итерации
- Исправьте вручную в n8n UI
- Запустите тестирование заново

### Ошибка: "Workflow ID не найден"

**Решение:**
- Добавьте Workflow ID в Issue body: `**Workflow ID:** \`workflow-id\``
- Или укажите как аргумент: `node fix_and_test_workflow.js 22 workflow-id`

### Ошибка: "GITHUB_TOKEN не установлен"

**Решение:**
```bash
export GITHUB_TOKEN="your-token"
```
