# Выполнение блока задач (Task Block Execution)

**Версия:** v1.0  
**Назначение:** Автоматическое выполнение блока задач (Epic/Issues) с тестированием и исправлением до конца

## 🎯 Что делает

Скрипт `execute_task_block.js` автоматически:
1. ✅ Определяет блок задач (Epic или список Issues)
2. ✅ **Если Epic/Issues не найдены — создаёт их автоматически**
3. ✅ Выполняет все задачи последовательно
4. ✅ Для каждой задачи:
   - Проверяет зависимости
   - Выполняет задачу
   - Если требует тестирования workflow — запускает цикл исправления и тестирования
   - Исправляет ошибки автоматически
   - Повторяет до успеха
   - **Проверяет чеклист — все пункты должны быть выполнены (`[x]`)**
   - **Проверяет тестирование — последний execution должен быть успешен**
   - **Только после прохождения чеклиста И тестирования переходит к следующей задаче**
5. ✅ Работает до конца (все задачи выполнены)

## 📋 Ручной запуск

```bash
# Выполнить Epic
node tests/scripts/execute_task_block.js 23

# Выполнить конкретные Issues

# Выполнить STEP (поиск по названию)
node tests/scripts/execute_task_block.js "STEP 04"
```

## 🔄 Процесс работы

```
1. Определение блока задач
   ↓
2. Проверка наличия Epic/Issues
   ↓
3. Если не найдены → создание Epic и Issues
   ↓
4. Для каждой задачи:
   ↓
5. Проверка зависимостей
   ↓
6. Выполнение задачи
   ↓
7. Если требует тестирования workflow:
   ↓
8. Цикл исправления и тестирования
   ↓
9. Повтор до успеха
   ↓
10. Проверка чеклиста (все пункты должны быть [x])
   ↓
11. Проверка тестирования (последний execution успешен)
   ↓
12. Только после прохождения чеклиста И тестирования:
   ↓
13. Переход к следующей задаче
   ↓
14. Работа до конца (все задачи выполнены)
```

## 💡 Преимущества

- ✅ **Автоматическое выполнение блока задач** — не нужно выполнять каждую задачу вручную
- ✅ **Тестирование и исправление** — автоматически для каждой задачи
- ✅ **Работа до конца** — все задачи выполняются последовательно
- ✅ **Проверка зависимостей** — автоматическая проверка перед выполнением
- ✅ **Интеграция с Cursor** — всё в одном месте

## 📋 Поддерживаемые типы задач

### ✅ Автоматически выполняются:

1. **Задачи с тестированием workflow:**
   - Автоматически запускается цикл исправления и тестирования
   - Исправляются ошибки автоматически
   - Повторяется до успеха
   - **Проверяется чеклист — все пункты должны быть выполнены**
   - **Проверяется тестирование — последний execution должен быть успешен**

2. **Задачи с зависимостями:**
   - Автоматически проверяются зависимости
   - Выполняются только после выполнения зависимостей
   - **Проверяется чеклист — все пункты должны быть выполнены**

### ⚠️ Критерии завершения задачи:

**Задача считается выполненной ТОЛЬКО если:**
- ✅ Все пункты чеклиста отмечены как выполненные (`[x]`)
- ✅ Тестирование пройдено (для задач с workflow — последний execution успешен)
- ✅ Все зависимости выполнены

### ⚠️ Требуют ручного выполнения:

1. **Задачи без workflow:**
   - Database Schema
   - Документация
   - Другие задачи без workflow

2. **Задачи с ошибками, которые нельзя исправить автоматически:**
   - Connection errors
   - Expression errors
   - Unknown errors

## 🔧 Troubleshooting

### Ошибка: "Зависимости не выполнены"

**Решение:**
- Выполните зависимости вручную
- Или запустите скрипт снова после выполнения зависимостей

### Ошибка: "Workflow ID не найден"

**Решение:**
- Добавьте Workflow ID в Issue body: `**Workflow ID:** \`workflow-id\``
- Или выполните задачу вручную

### Ошибка: "Задача не может быть выполнена автоматически"

**Решение:**
- Выполните задачу вручную
- После выполнения запустите скрипт снова

### Ошибка: "Чеклист не выполнен"

**Решение:**
- Обновите Issue, отметив все чекбоксы как выполненные (`[x]`)
- Запустите скрипт снова

### Ошибка: "Тестирование не пройдено"

**Решение:**
- Запустите workflow в n8n UI
- Убедитесь, что последний execution успешен
- Запустите скрипт снова
