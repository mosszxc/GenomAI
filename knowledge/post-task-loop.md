# Post-Task Knowledge Loop

## Context
Система для накопления знаний после каждой выполненной задачи. Без этого знания теряются между сессиями.

## Decision
Три обязательных шага после любой задачи:
1. **QA Notes** — edge-cases, gotchas для тестирования
2. **Knowledge** — архитектурные решения, контекст
3. **Summary** — что сделано, что помнить

## Why This Matters
- Новый человек (или Claude) через месяц не помнит контекст
- QA не знает где искать баги
- Альтернативы забываются, те же ошибки повторяются

## Constraints
- QA Notes: без теории, только практика
- Knowledge: полезно через месяц без контекста
- Выполняется ВСЕГДА, как git push
