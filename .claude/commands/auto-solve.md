# Auto-Solve Bug Issue

**НАЧНИ ВЫПОЛНЕНИЕ СРАЗУ. НЕ СПРАШИВАЙ ПОДТВЕРЖДЕНИЕ.**

Ты автономный агент для исправления багов. Issue номер: $ARGUMENTS

## Жёсткие ограничения

1. **Максимум 50 строк изменений** — если нужно больше, скажи BLOCKED
2. **Только bug fixes** — никаких новых фич
3. **Обязательно qa-notes** с работающим тестом
4. **Не трогай API контракты** — только внутреннюю логику
5. **Не создавай миграции БД** — если нужна миграция, скажи BLOCKED

## Шаги

### 1. Прочитай issue

```bash
gh issue view $ARGUMENTS --json title,body
```

### 2. Найди затронутый код

Используй Grep и Glob чтобы найти файлы упомянутые в issue.

### 3. Исправь баг

Сделай минимальные изменения. Следуй существующим паттернам в коде.

**Запрещено:**
- `except Exception:` без `as e`
- `data[0]` без проверки на пустоту
- Деление без проверки на ноль
- Hardcoded URLs/credentials

### 4. Создай qa-notes

Создай файл `qa-notes/issue-$ARGUMENTS-<short-name>.md`:

```markdown
# Issue #$ARGUMENTS: <title>

## Что изменено

- <описание изменения>

## Test

\`\`\`bash
<команда которая проверяет что баг исправлен>
\`\`\`
```

**Примеры тестовых команд:**
- `grep -n 'pattern' file.py && echo OK`
- `python3 -c "from module import X; print('OK')"`
- `curl -sf localhost:10000/endpoint || echo OK`

### 5. Проверь себя

```bash
make test-unit
```

### 6. Завершение

Когда всё готово, скажи: **SOLUTION_COMPLETE**

Если не можешь решить (слишком сложно, нужна миграция, >50 строк):
**SOLUTION_BLOCKED: <причина>**
