# Claude Code Workflow — Выжимка

> На основе практик соло-девов с 350k+ LOC кодовых баз, где 80%+ кода пишет Claude Code.

---

## Главный паттерн: GitHub Flow Loop

```
Plan → Create → Test → Deploy → /clear → Repeat
```

1. **Plan** — разбить фичу на гранулярные Issues
2. **Create** — Claude пишет код по Issue
3. **Test** — запустить тесты (автоматизировать!)
4. **Deploy** — merge PR
5. **Reset** — `/clear` и следующая задача

**Правило:** Ясные спеки → ясные коммиты.

---

## Три фазы работы над задачей

### Фаза 1: Исследование (Plan Mode — Shift+Tab×2)
```
> проанализируй архитектуру X
> найди где хранится Y
> объясни data flow от A до B
```

### Фаза 2: Планирование
```
> создай детальный план реализации
> какие риски и зависимости?
```

### Фаза 3: Реализация (Normal Mode)
```
> реализуй по плану
> напиши тесты
```

**Важно:** Без явной инструкции Claude сразу прыгает в код. Просить планировать сначала!

---

## Управление контекстом

| Команда | Когда использовать |
|---------|-------------------|
| `/clear` | Между задачами — свежий контекст лучше забитого |
| `/rename feature-x` | В начале работы — легко найти сессию потом |
| `/compact` | Когда контекст переполнен |
| `claude -c` | Продолжить последнюю сессию |
| `Shift+Tab` | Переключение режимов |

---

## Режимы работы (Shift+Tab для переключения)

| Режим | Индикатор | Когда |
|-------|-----------|-------|
| **Normal** | (по умолчанию) | Обычная работа с подтверждениями |
| **Accept All** | `⏵⏵ accept edits` | Доверенные задачи, рефакторинг |
| **Plan Mode** | `⏸ plan mode` | Исследование, планирование |

## CLAUDE.md — главный файл контекста

Автоматически включается в каждый разговор. Держать:

```markdown
# Project Overview
[Что делает проект]

## Команды
- `npm test` — тесты
- `npm run build` — сборка

## Архитектура
[Ключевые паттерны, data flow]

## Code Style
[Правила форматирования]

## Частые проблемы
- Проблема X → Решение Y
```

## Permission setup

Настроить auto-approve для частых команд в `.claude/settings.local.json`:
```
git, npm, docker, gh, supabase, n8n...
```

Убирает 90% диалогов подтверждения.

Хорошие тесты = можешь доверять Claude коммитить автономно.

## Быстрые команды

```bash
# Запуск
claude              # интерактивная сессия
claude -c           # продолжить последнюю
claude -p "query"   # одноразовый запрос (без сессии)
claude commit       # создать коммит

# В сессии
/clear              # очистить контекст
/cost               # расход токенов
/context            # визуализация контекста
?                   # все shortcuts
```

## Субагенты

Для специализированных задач (работают в отдельном контексте):
- **Explore** — быстрый поиск по кодовой базе
- **Plan** — планирование в Plan Mode
- **code-reviewer** — ревью кода

```
> используй Explore чтобы найти все обработчики ошибок
```

---

## Золотые правила

1. **Plan before code** — просить планировать перед реализацией
2. **Fresh context** — `/clear` между задачами
3. **Name sessions** — `/rename` в начале работы
4. **CLAUDE.md** — держать актуальным
5. **Automate tests** — чтобы доверять автономным коммитам
6. **Permission setup** — убрать лишние подтверждения
