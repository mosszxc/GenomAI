# Layer 1 — System Design / Logical Architecture

**Статус:** CANONICAL / ENFORCED  
**Приоритет:** Высокий (после Layer 0)

---

## Назначение

Layer 1 содержит **логическую архитектуру системы** — спецификации компонентов, схемы данных, контракты.

Эти документы:
- определяют **структуру данных** (Canonical Schema)
- описывают **логику принятия решений** (Decision Engine)
- фиксируют **контракты входных данных** (Data Contracts)
- определяют **взаимодействие компонентов**

**Layer 1 должен соответствовать Layer 0 (Doctrine).**

---

## Документы Layer 1

### 📐 [Canonical Schema Specification](./CANONICAL_SCHEMA.md) (v1.2)
Каноническая схема данных. Строгая типизация всех сущностей, enum определения, правила валидации.

### ⚙️ [Decision Engine Specification](./DECISION_ENGINE.md) (v1.1)
Детерминированное ядро системы. Фиксированный порядок проверок, правила принятия решений.

### 🔒 [Data Contracts Specification](./DATA_CONTRACTS.md) (v1.0)
Контракты входных данных. Правила валидации, обработка невалидных данных.

---

## Правила изменения

- Изменения должны быть **совместимы с Layer 0**
- Требуется **версионирование**
- Требуется **миграционный план** (если breaking changes)
- Должны быть **задокументированы** причины изменений

---

## Связь с другими слоями

- **Основан на:** Layer 0 (Doctrine)
- **Влияет на:** Layer 2 (Product & Integration), Layer 3 (Implementation)

**Layer 2 и Layer 3 должны соответствовать Layer 1.**
