# Output Payloads Specification
## GenomAI — System Output Payloads (Layer 2)

**Версия:** v1.0  
**Статус:** CANONICAL / LAYER 2  
**Приоритет:** Высокий  
**Основан на:** Usage Doctrine, TELEGRAM_INTERACTION_MODEL, DATA_FLOW, Decision Engine

---

## 1. Purpose

Данный документ определяет какие выходы система GenomAI имеет право отправлять пользователю, в каком логическом виде, и какие данные принципиально не раскрываются.

Документ:
- описывает продуктовые выходы, а не внутренние сущности,
- не описывает формат сообщений Telegram,
- не описывает реализацию или UI.

---

## 2. Core Output Principle

**Система сообщает результаты и факты, но не раскрывает внутреннюю логику, память или оценки.**

**Любой output:**
- является следствием решения системы,
- не инициируется пользователем,
- не подлежит ручной корректировке.

---

## 3. Supported Output Types

Система имеет право отправлять только следующие типы выходов.

---

## 4. Decision Result Payload

### 4.1 Назначение

Сообщить пользователю факт принятого решения.

### 4.2 Когда отправляется

- после обработки нового Creative,
- после обновления Idea,
- при повторной оценке (re-evaluation).

### 4.3 Содержит (логически)

- `decision_type`: approve / reject / defer / allow_with_constraints
- краткое описание причин (decision summary из Decision Trace)
- указание, что решение окончательное на текущий момент

### 4.4 Не содержит

- scores (novelty_score, similarity_score)
- confidence values (confidence_weight)
- fatigue levels
- cluster info (active_cluster_id, cluster_at_decision)
- ML-сигналы (raw float или buckets)
- внутренние метрики Decision Engine

**Критическое ограничение:**  
Decision summary содержит только факты из Decision Trace (decision_type, failed_check, dominant_constraint), но не числовые пороги или оценки.

---

## 5. Generated Transcript Payload

### 5.1 Назначение

Передать новые транскрипты для тестирования.

### 5.2 Когда отправляется

- только после APPROVE / ALLOW_WITH_CONSTRAINTS от Decision Engine,
- только как следствие Decision Engine,
- не по запросу пользователя.

### 5.3 Содержит

- один или несколько транскриптов,
- указание «для теста»,
- привязку к исходному Creative (неявно через tracker_id).

**Критическое ограничение:**  
Транскрипты содержат **ТОЛЬКО текст того, что говорит лицо в ролике**.  
**НЕ содержат:** визуалы, изображения, primary/description.

### 5.4 Ключевые ограничения

- транскрипты **не ранжируются**,
- транскрипты **не помечаются** как «лучше / хуже»,
- транскрипты **не оптимизируются** под метрику,
- транскрипты **не являются** «улучшенной версией старого».

---

## 6. Constraint Notice Payload

### 6.1 Назначение

Сообщить ограничения, наложенные системой.

### 6.2 Примеры ограничений

- ограниченный mutation scope (Hypothesis Factory),
- временный defer (Decision Engine),
- повышенный риск (Risk Budget),
- ограничение масштабирования (context_frame mismatch).

### 6.3 Важно

**Ограничения:**
- описываются качественно,
- не раскрывают числовых порогов,
- не раскрывают внутренних метрик.

**Примеры формулировок:**
- "Ограниченный scope мутаций"
- "Временная отсрочка решения"
- "Повышенный риск"
- "Несовместимый контекст"

**Не раскрывается:**
- конкретные значения fatigue levels,
- числовые пороги Decision Engine,
- confidence weights,
- ML scores.

---

## 7. Outcome Acknowledgement Payload

### 7.1 Назначение

Подтвердить получение performance-фактов.

### 7.2 Когда отправляется

- после успешной валидации Outcome,
- после проверки trust conditions.

### 7.3 Содержит

- подтверждение приёма,
- указание, будет ли Outcome использован для обучения (trusted / untrusted),
- не содержит интерпретаций.

### 7.4 Не содержит

- интерпретаций результатов,
- оценок качества,
- прогнозов,
- рекомендаций.

---

## 8. Error / Rejection Payload

### 8.1 Назначение

Уведомить о некорректных входных данных.

### 8.2 Типовые причины

- отсутствует tracker_id,
- недоступно видео,
- ошибка транскрипции,
- невалидный формат performance,
- нарушение Data Contract.

### 8.3 Поведение

- состояние системы не меняется,
- обучение не происходит,
- данные логируются для аудита.

### 8.4 Содержит

- тип ошибки (missing_tracker_id, inaccessible_video, transcription_failure, invalid_format),
- факт отказа,
- отсутствие изменений в системе.

---

## 9. Silent Output Semantics

**Отсутствие сообщений означает:**
- система работает,
- нет новых решений,
- либо решение defer,
- либо нет допустимых выходов.

**Молчание — валидный output.**

Система не обязана отправлять сообщения при каждом внутреннем действии.

---

## 10. Forbidden Output Types

**Система никогда не отправляет:**
- «лучший транскрипт» (ранжирование),
- рекомендации по ставкам,
- прогнозы ROI,
- внутренние метрики (scores, confidence, fatigue),
- confidence / fatigue / scores (числовые значения),
- объяснения в стиле LLM reasoning,
- варианты «на выбор» (интерактивный выбор),
- интерпретации performance,
- советы по оптимизации,
- **объяснения через environment** (например: «фб ныл», «аккаунт плохой», «поэтому не сработало»).

**Критическое правило:**  
Environment не является объяснением для пользователя.  
Environment — корректирующий фактор внутри системы, не раскрывается в output.

**Любая попытка таких выходов — архитектурное нарушение.**

---

## 11. Security & Authority Boundary

**Outputs:**
- не предоставляют рычагов управления,
- не позволяют реконструировать Decision Engine,
- не раскрывают состояние памяти,
- не дают привилегий пользователю.

**Authority всегда остаётся внутри системы.**

---

## 12. Evolution Rule

**Изменение выходных payload'ов допустимо, если:**
- не нарушает Usage Doctrine,
- не превращает систему в советчика,
- не раскрывает внутреннюю логику,
- не вводит интерактивность.

**Любые изменения:**
- версионируются,
- совместимы по смыслу,
- не требуют ручной корректировки пользователем.

---

## 13. Layer Boundary Notice

Этот документ:
- завершает Layer 2 (Product & Interaction),
- не влияет на Layer 0–1,
- является базой для Layer 3 (Implementation).

---

## 14. Final Rule

**Система говорит ровно столько, сколько нужно для действия — и ни словом больше.**
