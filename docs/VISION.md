# GenomAI Vision Document

**Дата:** 2026-01-13
**Статус:** Согласовано

---

## 1. Конечная цель

**GenomAI** — Autonomous Creative Decision System с **human-in-the-loop**.

Система автоматически анализирует креативы, обучается на рыночных данных и помогает байерам создавать эффективные креативы.

**Принцип: Market = Truth** — только данные трекера определяют что работает.

---

## 2. Два выхода системы

| Output | Для кого | Что даёт | Статус |
|--------|----------|----------|--------|
| **Готовые тексты** | Система (автогенерация) | Тексты креативов для автоматического тестирования | Будущее |
| **Дашборд меты** | Байеры | Аналитика трендов (как Dota2ProTracker) | Приоритет |

---

Байер — **учитель системы**, не просто потребитель.

## 4. Архитектурные решения

### 4.1 Decision Engine

| Контекст | Роль DE | Поведение |
|----------|---------|-----------|
| **Автогенерация** | Обязателен | APPROVE/REJECT/DEFER — фильтр плохих гипотез |
| **Байеры** | Advisory | Показывает warnings, но не блокирует |

**8 проверок остаются**, но применяются по-разному:
- Для автогена: все 8 как hard constraints
- Для байеров: Death Memory и Fatigue как warnings

### 4.2 Learning Loop

Учится на **трёх уровнях**:

| Уровень | Что отслеживает | Как используется |
|---------|-----------------|------------------|
| **Компоненты** | Win rate по модулям | Рекомендации "что работает" |
| **Идеи** | Confidence, Fatigue, Death | Фильтрация в DE |
| **Кластеры** | Saturation, Diversity | Exploration vs Exploitation |

### 4.3 Рекомендации для байеров

**Формат: Дашборд меты**

Показывает:
- **HOT** — что работает сейчас (высокий win rate)
- **COLD** — что устало (high fatigue, declining trend)
- **GAPS** — где мало тестов (underexplored)
- **CPA по модулям** — средний CPA для каждого типа
- **История** — динамика по неделям
- **Сравнение гео** — что работает в DE vs MX vs US

## 5. Модули (7 независимых переменных)

### Проблема исходной схемы

Canonical Schema v2 содержит ~30 переменных, многие из которых **коррелируют** (мультиколлинеарность):
- `angle_type` ≈ `emotional_trigger` ≈ `emotion_primary`
- `opening_type` ≈ `hook_mechanism`
- `message_structure` ≈ `story_type`

**Проблема**: Трудно понять что реально влияет на результат, win rate одной переменной может быть артефактом другой.

### Решение: Редукция до 7 независимых переменных (MVP)

| # | Переменная | Что измеряет | Убраны (коррелируют) |
|---|------------|--------------|---------------------|
| 1 | `hook_mechanism` | Как зацепить | opening_type, hook_stopping_power |
| 2 | `angle_type` | Эмоциональный угол | emotional_trigger, emotion_primary, emotion_intensity |
| 3 | `message_structure` | Структура нарратива | story_type, story_bridge_present |
| 4 | `ump_type` | Уникальный механизм | ums_type, paradigm_shift_type |
| 5 | `promise_type` | Тип обещания | state_before, state_after |
| 6 | `proof_type` | Тип доказательства | proof_source, social_proof_pattern |
| 7 | `cta_style` | Призыв к действию | risk_reversal_type |

### Принцип выбора

- **Одна ведущая переменная** из каждой группы коррелированных
- Остальные переменные из Canonical Schema v2 **сохраняются в decomposed_creatives**, но не используются для learning
- **Расширяемость**: можем добавить `core_belief`, `context_frame`, `specificity_level` позже

### Использование переменных

| Слой | Какие переменные | Как используются |
|------|------------------|------------------|
| **Learning** | Все 7 | Win rate, Fatigue tracking |
| **Генерация** | hook_mechanism, angle_type, message_structure, ump_type | Комбинируются для новых текстов |
| **Аналитика** | Все 7 | Дашборд трендов для байеров |

## 10. Принципы (неизменные)

1. **Market = Truth** — только метрики трекера определяют успех
2. **Determinism** — одинаковый input → одинаковый output
3. **Schema First** — проверь колонки перед кодом
4. **Human-in-the-loop** — байер учит систему, система помогает байеру
5. **7 модулей** — независимые измерения креатива
