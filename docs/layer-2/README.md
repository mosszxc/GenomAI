# Layer 2 — Product & Integration Specs

**Статус:** В разработке  
**Приоритет:** Средний

---

## Назначение

Layer 2 содержит **спецификации продукта и интеграций** — API, интерфейсы, интеграционные контракты.

Эти документы:
- определяют **API endpoints** и протоколы
- описывают **интеграции** с внешними системами
- фиксируют **форматы данных** для обмена
- определяют **поведение продукта** на уровне пользователя

**Layer 2 должен соответствовать Layer 0 и Layer 1.**

## Правила изменения

- Изменения должны быть **совместимы с Layer 0 и Layer 1**
- Требуется **версионирование API**
- Breaking changes требуют **миграционного плана**
- Должны быть **задокументированы** примеры использования
