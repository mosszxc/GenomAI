# Layer 0 — Doctrine / Конституция системы

**Статус:** CANONICAL / ENFORCED  
**Приоритет:** Высший

---

## Назначение

Layer 0 содержит **доктринальные документы** — фундаментальные принципы, правила и политики системы GenomAI.

Эти документы:
- определяют **что система есть** и **что она не есть**
- фиксируют **неизменяемые принципы** работы
- устанавливают **границы допустимого**
- описывают **модель взаимодействия** с системой

**Эти документы выше всех остальных слоев.**  
**Нарушение Layer 0 считается архитектурным дефектом.**

---

## Документы Layer 0

### 🔒 [Architecture Lock / Doctrine](./ARCHITECTURE_LOCK.md) (v1.0)
Высший архитектурный артефакт проекта. Определяет инварианты, запреты и роли компонентов.

### 📘 [Concept Document](./CONCEPT.md) (v1.0)
Концептуальный документ проекта. Общее понимание системы, цели, принципы, процессы.

### 📚 [Domain Model / Glossary](./DOMAIN_MODEL.md) (v1.0)
Единый словарь предметной области. Определение ключевых сущностей и терминов.

### 🤖 [LLM Usage Policy](./LLM_USAGE_POLICY.md) (v1.0)
Политика использования LLM. Где, как и для чего разрешено использовать языковые модели.

### 🧠 [Learning & Memory Policy](./LEARNING_MEMORY_POLICY.md) (v1.2)
Политика обучения и памяти. Как система обучается, что запоминает, как знания устаревают.

### 👤 [Usage & Interaction Doctrine](./USAGE_DOCTRINE.md) (v1.1)
Модель взаимодействия медиабайера с системой. Push-модель, запрещённое использование.

---

## Правила изменения

- Изменения допускаются **редко**
- Только через **отдельное архитектурное решение**
- Все изменения **версионируются** и **логируются**
- Требуется **явное одобрение**

---

## Связь с другими слоями

- **Layer 1** (System Design) должен соответствовать Layer 0
- **Layer 2** (Product & Integration) должен соответствовать Layer 0 и Layer 1
- **Layer 3** (Implementation) должен соответствовать всем вышестоящим слоям

**Если Layer 1+ противоречит Layer 0 — Layer 0 прав.**
