# Usage & Interaction Doctrine
## GenomAI — Media Buyer Interaction Model

**Версия:** v1.1  
**Статус:** CANONICAL / ENFORCED  
**Приоритет:** Ниже Architecture Lock, выше UI и задач

---

## 1. Purpose

Данный документ определяет как продукт GenomAI используется медиабайерами,  
и строго фиксирует допустимую модель взаимодействия человека с системой.

Документ:
- описывает реальные сценарии эксплуатации продукта,
- фиксирует коммуникационный контур,
- предотвращает превращение системы в:
  - генератор текстов,
  - LLM-оптимизатор,
  - ручной decision-support инструмент.

Документ **не описывает UI, UX или интерфейсные детали**,  
а описывает **архитектурно допустимую форму использования**.

---

## 2. Primary User

### Media Buyer (основной пользователь)

Медиабайер использует GenomAI как:
- внешний стратегический мозг,
- систему дисциплины тестирования,
- источник новых транскриптов для теста,
  сгенерированных на основе решений системы.

Медиабайер:
- **не управляет** логикой принятия решений,
- **не взаимодействует** с системой в режиме диалога,
- **не «спрашивает»** систему, что делать.

---

## 3. Communication Channel (начальный этап)

### Telegram Bot (initial interface)

На первоначальном этапе всё взаимодействие с GenomAI осуществляется через Telegram-бота.

Telegram-бот:
- является **транспортным интерфейсом**, а не логическим компонентом,
- **не содержит** бизнес-логики,
- **не принимает** решений,
- **не интерпретирует** данные.

Telegram используется исключительно как:
- канал загрузки входных данных,
- канал доставки выходов системы,
- канал уведомлений.

**Смена канала (UI, web, dashboard) не влияет на архитектуру системы  
и не меняет Usage Doctrine.**

---

## 4. Allowed User Actions (push-модель)

Медиабайер **не запрашивает** действия системы.  
Он может только:

### Передавать входные данные:
- транскрипты креативов (internal / spy),
- фактический performance-фидбек по запускам.

### Получать выходы системы:
- решения Decision Engine (approve / reject / defer),
- decision trace (причины),
- ограничения (constraints),
- новые транскрипты для теста, сгенерированные системой.

**Коммуникация носит односторонний и событийный характер:**
```
вход → обработка → выход
```
**без интерактивных запросов.**

---

## 5. Core Usage Loop (взгляд медиабайера)

```
[Загрузка транскриптов]
            ↓
[Запуск / рынок]
            ↓
[Передача performance-фактов]
            ↓
[Ответ системы:
 решения + новые транскрипты]
```

**Ключевой принцип:**  
Медиабайер кормит систему фактами,  
система возвращает решения и допустимые реализации.

---

## 6. How New Transcripts Are Produced

### 6.1 Что система обучает

GenomAI обучается не на текстах, а на:
- сочетаниях канонических переменных:
  - angle_type
  - core_belief
  - promise_type
  - emotion
  - state_transition
  - context_frame
- последствиях решений во времени.

### 6.2 Что означает «система выдала транскрипт»

Выдача нового транскрипта означает:

**Decision Engine:**
- допустил Idea (или группу идей),
- определил ограничения (risk, fatigue, horizon).

**Hypothesis Factory:**
- сгенерировала транскрипт
- как реализацию разрешённой идеи
- в рамках допустимого mutation scope.

**Транскрипт:**
- не является «улучшенной версией старого»,
- не оптимизируется напрямую под метрику,
- не считается рекомендацией «лучшего текста».

### 6.3 Что система НИКОГДА не делает

Система:
- **не принимает** запросы «сделай лучше»
- **не оптимизирует** текст по CTR / CPA
- **не подмешивает** performance в промпты LLM
- **не генерирует** тексты вне разрешённых идей
- **не действует** в режиме чата

---

## 7. Feedback Contract

Медиабайер передаёт системе **только факты:**
- spend
- primary metric
- контекст запуска
- временные границы

**Запрещено:**
- передавать интерпретации,
- помечать идеи как «рабочие»,
- объяснять системе «почему не зашло».

**Система обучается исключительно на реакции рынка.**

---

## 8. Forbidden Usage (blacklist)

**Запрещено:**
- Использовать систему как копирайтера
- Ожидать интерактивного диалога
- Ручным образом выбирать идеи
- Редактировать fatigue / death / confidence
- Использовать Telegram-бот как интерфейс управления логикой

**Любое из вышеперечисленного — нарушение Usage Doctrine.**

---

## 9. Responsibility Split

| Вопрос | Ответственный |
|--------|---------------|
| Какие идеи допустимы | Decision Engine |
| Какие транскрипты сгенерированы | Hypothesis Factory |
| Какой текст «лучше» | Рынок |
| Почему идея отклонена | Decision Trace |
| Запуск и интерпретация | Media Buyer |
| Вмешательство | Только override |

---

## 10. Final Rule

**Если использование GenomAI:**
- превращает систему в чат,
- делает медиабайера оператором решений,
- или смещает оптимизацию на уровень текста,

**— это считается архитектурным дефектом продукта.**
